Padding Oracle Attack Notes

For Mere Mortals

Published Sun, May 17, 2020 by Anthony Towry

Photo by dylan nolte on Unsplash - ::cropped::

I’m not too proud to admit that I’ve let a padding oracle vulnerability go unexploited in the past. Time had run out and it was documented and easily patched without the need for a PoC.

Well, I ran into this on a CTF challenge recently and this time I was determined to see something work.

I first tried mocking something up in Ruby using Poracle. Ultimately, I didn’t make this happen, but the blog posts that are linked in the README.md were awesome resources.

Now, I’m old enough to remember the .Net framework padding oracle vulnerability from about a decade ago which I alluded to earlier. In those days, we had padbuster.pl, which I had completely forgotten. Luckily, landing on this video brought it back.

I tried giving it information in the -error flag and directing it to -usebody, but this additional information was a bit of an overthink and didn’t help the situation. Stripping it down to the basics lets padBuster do it’s own response analysis.

perl padBuster.pl http://target/?post=<cryptoadstuff> <cryptoadstuff> 16

This worked perfectly! A few minutes later I had the decrypted values for the challenge. Dead simple is sometimes the way to go.

Now when I run into this in another 10 years, I’ll have these notes.
