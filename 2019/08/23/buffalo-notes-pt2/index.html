Playing With Buffalo - Part 0.3

Notes

Published Fri, Aug 23, 2019 by Tony Towry

All notes are referencing buffalo 0.14.8 on Ubuntu.

Using buffalo generate resource myresourcename generated an actions file with an undefined: T i18n.Translator in the flash message template. No big deal and the generators are still pretty delightful.

No Templates Box Defined

So, out of the gate I was receiving an error that no TemplatesBox was defined when making a request to /. At first I thought I needed to read up on templating and maybe I had screwed up the default index.html. As it turns out, thanks to this post Web Apps With Buffalo - Part 1, I found that I was missing a render.go option that lets buffalo know where all of the templates are stored.

The fixed init in render.go looked like this: 

func init() {
    r = render.New(render.Options{
        TemplatesBox: packr.NewBox(“../templates”),
    })
}

0.14.10 - Critical Security Fix

The release notes for 0.14.10 include dependency updates for packr that fix a directory traversal issue in the way assetsBox works. You might want to update if you’ve got assets served by buffalo.
